# SiteForge — Client Setup Checklist

> **This file lives at `docs/CLIENT_SETUP_CHECKLIST.md` in the repo.**
> Use this as a quick-reference for every new client. Organized by feature.
> For detailed explanations, see `docs/NEW_CLIENT_GUIDE.md`.

---

## 1. Scaffold the Client (2 minutes)

```bash
yarn new-client
```

Follow the prompts. This creates everything: config, theme, env file, directories, scripts, registry.

After the CLI finishes, verify:
- [ ] `clients/{name}/` folder exists with config.ts, theme.ts, content/
- [ ] `env/.env.{name}` exists
- [ ] `yarn dev:{name}` script exists in package.json
- [ ] Client registered in `src/lib/client-config.ts`

---

## 2. Per-Feature Setup

Each feature needs specific accounts/keys. Only do the sections below for features you enabled.

---

### Blog + CMS (features.blog = true)

**Needs: GitHub token, admin credentials, Cloudflare R2**

#### GitHub Token (same token for all clients)
If you already have a token from another client, reuse it. Otherwise:

1. Go to https://github.com/settings/tokens
2. Generate new token (classic)
3. Scope: `repo` only
4. Copy the token

```
# env/.env.{name}
GITHUB_TOKEN=ghp_your_token_here
GITHUB_REPO=SergiuAlexandruPop/siteforge
GITHUB_BRANCH=main
```

#### Admin Credentials
Pick a strong password. Session secret was auto-generated by the CLI.

```
# env/.env.{name}
ADMIN_PASSWORD=pick_a_strong_password_here
ADMIN_SESSION_SECRET=already_generated_by_cli
```

#### Cloudflare R2 (for blog images)
R2 Account ID and API keys are **shared across all clients** — only create them once. Each client needs a **new bucket**.

**First time only — create API token:**
1. https://dash.cloudflare.com → R2 Object Storage
2. Right sidebar → Manage R2 API Tokens → Create API Token
3. Permissions: Object Read & Write, All buckets
4. Save the Access Key ID and Secret Access Key (shown once!)
5. Account ID is in your Cloudflare dashboard URL

**Per client — create bucket:**
1. R2 → Create Bucket
2. Name: `siteforge-{client-name}-images`
3. Location: Europe (WEUR)
4. Click on bucket → Settings → Public Access → Allow Access
5. Copy the public URL (`https://pub-xxx.r2.dev`)

```
# env/.env.{name}
R2_ACCOUNT_ID=your_account_id            # Same for all clients
R2_ACCESS_KEY_ID=your_access_key         # Same for all clients
R2_SECRET_ACCESS_KEY=your_secret_key     # Same for all clients
R2_BUCKET_NAME=siteforge-{name}-images   # Unique per client
R2_PUBLIC_URL=https://pub-xxx.r2.dev     # Unique per client (from bucket settings)
```

### Contact Form (features.contactForm = true)

**Needs: Resend API key**

**Option A — Shared account (quick, free):**
1. https://resend.com/api-keys
2. Create API key (name: client name)
3. Emails send from `onboarding@resend.dev`

**Option B — Custom domain (professional):**
1. Create separate Resend account (1 free domain per account)
2. Add client domain → add DNS records shown by Resend
3. Wait for verification (~5 min)
4. Create API key

```
# env/.env.{name}
RESEND_API_KEY=re_xxxxxxxxxxxx
RESEND_FROM_EMAIL=noreply@clientdomain.ro    # or onboarding@resend.dev
RESEND_TO_EMAIL=client@email.ro              # Where messages arrive
```

---

### Smartsupp Chat (features.smartsupp = true)

**Needs: Smartsupp account + ID**

1. Client creates account at https://www.smartsupp.com (free: 1 agent)
2. Dashboard → Settings → Chat box → Get chat code
3. Find the ID in the script snippet (long alphanumeric string)

```
# env/.env.{name}
NEXT_PUBLIC_SMARTSUPP_ID=xxxxxxxxxx
```

---

### i18n (features.i18n = true)

**Needs: nothing extra.** Just add content files:
- `clients/{name}/content/pages-en/index.md` — English homepage
- `clients/{name}/content/pages-en/about.md` — etc.
- `clients/{name}/content/blog-en/` — English blog posts

The language toggle appears automatically. Missing pages show "Not available in English" placeholder.

---

### Dark Mode (features.darkMode = true)

**Needs: nothing extra.** Toggle appears automatically. Customize colors in `clients/{name}/theme.ts` under `colors.dark`.

---

### Google Analytics (optional, not a feature flag)

**Needs: GA4 property**

1. https://analytics.google.com → Admin → Create Property
2. Property name: client's site name
3. Create Web data stream → enter domain
4. Copy Measurement ID (starts with `G-`)

```
# env/.env.{name}
NEXT_PUBLIC_GA4_ID=G-XXXXXXXXXX
```

## 4. Deploy to Vercel

### 4.1 Create Vercel Project

1. https://vercel.com → Add New Project
2. Import the `siteforge` GitHub repository
3. **Before deploying, configure:**
   - Project Name: `siteforge-{client-name}`
   - Framework: Next.js (auto-detected)
   - Root Directory: `.`
   - Build Command: `yarn build:{client-name}`
   - Output Directory: (leave blank)

### 4.2 Add Environment Variables

Go to Settings → Environment Variables. Copy EVERY non-empty value from `env/.env.{name}`:

**Always required:**

| Variable | Environment | Notes |
|---|---|---|
| `ACTIVE_CLIENT` | Production + Preview | Must match client folder name |

**If blog enabled:**

| Variable | Environment | Notes |
|---|---|---|
| `GITHUB_TOKEN` | Production | Your GitHub PAT |
| `GITHUB_REPO` | Production | `user/siteforge` |
| `GITHUB_BRANCH` | Production | `main` |
| `ADMIN_PASSWORD` | Production | Strong password |
| `ADMIN_SESSION_SECRET` | Production | Random 32-char string |
| `R2_ACCOUNT_ID` | Production + Preview | Cloudflare account |
| `R2_ACCESS_KEY_ID` | Production + Preview | R2 API key |
| `R2_SECRET_ACCESS_KEY` | Production + Preview | R2 API secret |
| `R2_BUCKET_NAME` | Production + Preview | Client's bucket name |
| `R2_PUBLIC_URL` | Production + Preview | Client's bucket public URL |

**If contact form enabled:**

| Variable | Environment | Notes |
|---|---|---|
| `RESEND_API_KEY` | Production | Resend API key |
| `RESEND_FROM_EMAIL` | Production | Sender address |
| `RESEND_TO_EMAIL` | Production | Client's email |

**If Smartsupp enabled:**

| Variable | Environment | Notes |
|---|---|---|
| `NEXT_PUBLIC_SMARTSUPP_ID` | Production + Preview | Smartsupp widget ID |

**If Analytics configured:**

| Variable | Environment | Notes |
|---|---|---|
| `NEXT_PUBLIC_GA4_ID` | Production | GA4 Measurement ID |

### 4.3 Deploy

Click Deploy. Wait ~60-90 seconds. Verify at the preview URL.

---

## 5. Connect Domain

### 5.1 Add in Vercel
Vercel → Project → Settings → Domains → Add:
- `clientdomain.ro`
- `www.clientdomain.ro` (redirects to root)

### 5.2 Update DNS
Add these records at the domain registrar:

| Type | Name | Value |
|---|---|---|
| A | `@` | `76.76.21.21` |
| CNAME | `www` | `cname.vercel-dns.com` |

**Where to add DNS records:**
- Namecheap: Dashboard → Domain → Advanced DNS
- GoDaddy: Domain → DNS Management
- Other: look for "DNS settings" or "DNS records"

### 5.3 Verify
- Wait 5-30 minutes (check at https://dnschecker.org)
- Visit `https://clientdomain.ro` — should load with padlock
- Visit `http://clientdomain.ro` — should redirect to HTTPS

## Quick Reference: Adding Features Later

If a client wants to enable a feature that was initially off:

```bash
yarn toggle-feature
```

This shows current features, lets you enable/disable, tells you exactly which env vars are needed and how to get them, and updates config.ts automatically.

After toggling:
1. Fill in any missing env vars in `env/.env.{name}`
2. Update the same vars in Vercel (Settings → Environment Variables)
3. Redeploy on Vercel (push to main or trigger manual deploy)
